# 文化祭 採点・集計アプリ

学校の端末（PC・タブレット）から採点し、偏りを補正した集計で順位を出すアプリです。
集計結果は審査に影響しないよう、パスワードを知る担当者のみが閲覧できます。

## 特長

- **点数段階を自由に変更**（例: 1〜5、1〜10、刻みも設定可能）
- **審査員 / 来場者を分けて集計**
- **複数の採点項目**（重み付け可）に対応
- **票の偏りを補正する集計方法を切り替え可能**
- **順位表示** と、集計方法ごとの順位比較
- **結果はパスワード保護**。投票端末には結果画面を一切表示しない

## 集計方法（管理画面で切替）

| 方法 | 何を補正するか |
|------|----------------|
| 単純合計 | なし。票が多い出し物ほど有利 |
| 平均 | 票数のばらつき |
| 正規化平均（偏差） | **審査員ごとの甘辛**（厳しい/甘い採点）を標準化して補正 |
| ベイズ平均 | 票数が少ない出し物を全体平均へ引き寄せ、偶然の高評価を抑制 |
| トリム平均 | 上下の外れ値を除外 |

> 「全員が全てに投票するわけではない」ため単純合計だけでは歪みます。
> 審査員の甘辛が気になるなら **正規化平均**、少数票の偶然が気になるなら **ベイズ平均** が有効です。

## 公開版（Vercel）

本番は Vercel にデプロイ済みで、どの回線からでも同じURLでアクセスできます。

- 投票画面: `https://bunkasai-app.vercel.app/` （QRコードはこのURLで作成）
- 管理・結果: `https://bunkasai-app.vercel.app/admin.html`

票・設定は Upstash Redis（Vercel連携の無料DB）に保存されるため、再デプロイしても消えません。
コードを push して再デプロイするには:

```
npx vercel deploy --prod
```

Vercel版で config.json を変更したい場合は、編集して再デプロイ後、
管理画面「設定 → config.json を再読み込み」を押すと反映されます
（管理画面の各タブから直接編集する方が簡単です）。

## 起動方法（ローカル / 校内LAN運用）

1. Node.js（18以上）をインストール
2. このフォルダで初回のみ:
   ```
   npm install
   ```
3. サーバー起動:
   ```
   npm start
   ```
4. ブラウザでアクセス:
   - 投票画面: `http://localhost:3000/`
   - 管理・結果: `http://localhost:3000/admin.html`（初期パスワード: `admin`）

### 他の端末から使う（複数端末で共有）

1. サーバーを動かすPCのIPアドレスを調べる（Mac: `ipconfig getifaddr en0`）
2. 各端末のブラウザで `http://<そのIP>:3000/` を開く（同じWi-Fi/LAN内）
3. **管理URL（/admin.html）とパスワードは担当者にだけ共有**してください

## config.json（マスターデータの直接編集）

出し物・採点項目・点数段階・タイトルは、管理画面のほかに
リポジトリ直下の `config.json` を直接編集しても変更できます。

```json
{
  "title": "文化祭 採点",
  "scale": { "min": 1, "max": 5, "step": 1 },
  "criteria": [
    { "id": "c1", "name": "企画性", "weight": 1 }
  ],
  "items": [
    { "id": "i_xxx", "name": "1年A組 お化け屋敷", "description": "教室展示" }
  ]
}
```

- `id` は省略可（起動時に自動で振られます）。**一度振られた id は変えないでください**（票との紐付けが切れます）
- 編集後は管理画面「設定 → データ管理 → config.json を再読み込み」を押すか、サーバーを再起動
- 票データは `data/db.json` に分かれて保存されるため、config.json の編集で票は消えません

## 審査員の本人確認

審査員は投票画面で名前を記入し、**審査員コード**（担当者から伝えられた6桁の合言葉）を
入力して採点を開始します。コードはサーバー側で検証されるため、コードを知らない来場者は
審査員として採点できません。同じ名前の採点は同一審査員として集計されます。

## 運用の流れ

1. 管理画面にログイン → 「設定」で点数段階・タイトルを調整
2. 「採点項目」を登録（例: 企画性・完成度・盛り上がり）
3. 「出し物」を登録
4. 投票端末は `http://<IP>:3000/` を開き、区分（審査員/来場者）を選んで採点
5. 集計担当は管理画面「結果・順位」で集計方法を選び、順位を確認
6. 本番前のテスト票は「設定 → データ管理 → 全投票をリセット」で消去

## セキュリティ・注意点

- **審査員コード**: 審査員として採点するには合言葉（初回起動時に自動生成される6桁コード）が必要です。管理画面「設定」で確認・変更し、審査員のかたにだけ伝えてください。来場者が審査員になりすますことを防ぎます（サーバー側でも検証します）
- 初回起動後、必ず「設定 → パスワード変更」で初期パスワードを変えてください
- 投票端末には管理URLを開かない・共有しないでください
- データは `data/db.json` に保存されます。バックアップはこのファイルをコピー

## データのエクスポート

管理画面「設定 → データ管理 → 投票データをCSV出力」で、全票を CSV（Excel対応）で書き出せます。
